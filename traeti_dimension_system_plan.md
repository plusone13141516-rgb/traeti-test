# TRAETI (TRAE Type Indicator) 维度系统设计方案

## 1. 概述
基于 TRAE 中文论坛用户（主要为开发者、程序员、AI 编程工具早期采用者）的真实画像，我们重新设计了这套人格/编码风格评估系统，简称为 **TRAETI**。
本方案将原有的情感/社交心理维度，全面替换为**代码习惯、问题解决、社区交互、技术态度、工具折腾** 5 大模型，共 15 个维度。

---

## 2. 维度分组与命名 (Dimension Meta)

我们将 15 个维度分为 5 个核心模型，每个模型包含 3 个子维度。

### 模型一：代码习惯模型 (Code Habit) - 标识: `C1, C2, C3`
- **C1: AI依赖度** (纯手敲 vs 极度依赖AI)
- **C2: 代码洁癖** (能跑就行 vs 极致优雅)
- **C3: 架构偏好** (单文件梭哈 vs 过度设计)

### 模型二：问题解决模型 (Problem Solving) - 标识: `P1, P2, P3`
- **P1: 排错首选** (查官方文档 vs 论坛/AI发问)
- **P2: 报错耐受度** (见红就慌 vs 稳如老狗)
- **P3: 探索深度** (CV工程师/贴代码就跑 vs 扒源码探究原理)

### 模型三：社区交互模型 (Community Interaction) - 标识: `I1, I2, I3`
- **I1: 活跃倾向** (万年潜水员 vs 论坛大水怪)
- **I2: 开源分享欲** (敝帚自珍 vs 狂热布道师)
- **I3: 讨论风格** (纯技术理性 vs 情绪输出/玩梗)

### 模型四：技术态度模型 (Tech Attitude) - 标识: `T1, T2, T3`
- **T1: 尝鲜阈值** (稳定至上保守派 vs 永远用 Beta 版的先锋)
- **T2: 效率狂热度** (随性开发 vs 极致压榨每一秒)
- **T3: 技术信仰** (实用主义/搞钱要紧 vs 技术极客/理想主义)

### 模型五：工具折腾模型 (Tool Usage) - 标识: `U1, U2, U3`
- **U1: 快捷键依赖** (鼠标流 vs 键盘流)
- **U2: 插件堆叠度** (原生极简 vs 插件装配机)
- **U3: 外观定制欲** (默认黑白主题 vs 花里胡哨自定义)

## 4. 维度顺序配置规则 (dimensionOrder)

在 `data.js` 中，必须严格定义这 15 个维度的计算顺序。所有的 `NORMAL_TYPES` (人格特征向量) 字符串（如 `HHH-MML-LLL-HMH-LHL`）必须与此数组的顺序保持一致。

```javascript
// TRAETI 维度的标准顺序配置
const dimensionOrder = [
  'C1', 'C2', 'C3',   // 1-3: 代码习惯
  'P1', 'P2', 'P3',   // 4-6: 问题解决
  'I1', 'I2', 'I3',   // 7-9: 社区交互
  'T1', 'T2', 'T3',   // 10-12: 技术态度
  'U1', 'U2', 'U3'    // 13-15: 工具折腾
];
```

**配置说明：**
当您在配置新人格（例如：`CTRL-C 工程师`）的 `pattern` 时，若写下 `LHM-HLL-MML-LHH-LHL`，代表：
- **C1(AI依赖度)** = L, **C2(代码洁癖)** = H, **C3(架构偏好)** = M
- 依次类推，第 N 个字符必须对应 `dimensionOrder` 数组的第 N 个维度。

## 5. 替换指南 (Replacement Guide)

拿到本设计方案后，您可以按照以下步骤将 TRAE 论坛定制版数据写入您的项目：

1. 打开 `data.js`。
2. 将原有的 `dimensionMeta` 替换为本文档第二节的内容（注意转换为对应的 JSON 对象格式）。
3. 将本文档第三节的 `DIM_EXPLANATIONS` 代码块直接复制并覆盖 `data.js` 中的同名常量。
4. 将本文档第四节的 `dimensionOrder` 数组覆盖原有的数组。
5. 接下来，您可以基于这 15 个新维度，去构建专门针对程序员的 `questions`（题库）以及 `TYPE_LIBRARY` / `NORMAL_TYPES`（例如：CV 工程师、架构大师、Vim 狂魔、潜水白嫖怪等全新人格）。
